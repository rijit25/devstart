🏗️ Architecte de Données (DDL)

La base de tout : Créer des tables solides avec les bonnes contraintes.

-- Création de la table 'utilisateurs' CREATE TABLE utilisateurs ( id INT PRIMARY AUTO_INCREMENT, email VARCHAR(100) , nom VARCHAR(50) NULL, created_at DATETIME DEFAULT ); -- Création de la table 'commandes' avec Clé Étrangère CREATE TABLE commandes ( id INT PRIMARY KEY AUTO_INCREMENT, user_id INT, FOREIGN KEY (user_id) utilisateurs(id) );

🔍 L'Art de la Recherche (SELECT)

Filtrer, trier et limiter les données.

-- 1. Récupérer TOUTES les colonnes * clients; -- 2. Filtrer par ville SELECT nom FROM clients ville = 'Paris'; -- 3. Trier par prix décroissant SELECT * FROM produits BY prix ; -- 4. Recherche floue (commence par 'Dup') SELECT * FROM clients WHERE nom 'Dup%';

✍️ Gestion de Données (Insert, Update, Delete)

Modifier les données existantes.

-- 1. AJOUTER un nouvel utilisateur INTO utilisateurs (nom, age) ('Thomas', 25); -- 2. MODIFIER l'email utilisateurs email = 'new@mail.com' WHERE id = 42; -- 3. SUPPRIMER une ligne FROM commandes WHERE statut = 'annule';

🔗 Les Jointures (Relations)

Lier les tables entre elles.

-- 1. Intersection stricte (Clients AVEC Commandes) SELECT utilisateurs.nom, commandes.montant FROM utilisateurs JOIN commandes utilisateurs.id = commandes.user_id; -- 2. Tout garder à gauche (Clients même SANS Commandes) SELECT utilisateurs.nom, commandes.id FROM utilisateurs JOIN commandes ON utilisateurs.id = commandes.user_id;

🚀 Expert : Sous-requêtes & Dates

Des requêtes complexes pour des besoins réels.

-- 1. Trouver les produits commandés par 'Jean' (Sous-requête) SELECT * FROM produits WHERE id ( SELECT produit_id FROM commandes WHERE client = 'Jean' ); -- 2. Trouver les commandes des 7 derniers jours (Fonctions Date) SELECT * FROM commandes WHERE date_com > (NOW(), INTERVAL 7 DAY); -- 3. Combiner Nom et Prénom dans une colonne (Fonctions String) SELECT (nom, ' ', prenom) AS nom_complet FROM clients; -- 4. Compter par catégorie (Agrégation) SELECT cat, COUNT(*) FROM produits BY cat COUNT(*) > 10;

🛡️ Admin & Performance

Gérez les utilisateurs, les droits et optimisez la vitesse.

-- 1. CRÉER un utilisateur 'stagiaire' avec mot de passe CREATE USER 'stagiaire'@'localhost' BY 'motdepasse'; -- 2. DONNER le droit de lire (SELECT) sur toutes les tables GRANT ON *.* 'stagiaire'@'localhost'; -- 3. RETIRER tous les droits (Sécurité) REVOKE ALL PRIVILEGES 'stagiaire'@'localhost'; -- 4. CRÉER un INDEX pour accélérer les recherches sur le 'nom' CREATE idx_nom ON clients(nom); -- 5. CRÉER une VUE (Requête enregistrée) des clients parisiens CREATE vue_paris AS SELECT * FROM clients WHERE ville = 'Paris';
